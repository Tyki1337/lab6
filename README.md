## Ответы на вопросы

### **1. Что делает `fetch`?**

`fetch` — это встроенная функция JavaScript, которая отправляет HTTP-запросы на сервер. Она позволяет получать данные из внешних API, отправлять формы, загружать файлы и взаимодействовать с веб-ресурсами.

Основные свойства:

* работает асинхронно,
* возвращает **Promise**,
* позволяет получать данные в форматах JSON, текст, изображения, Blob и др.

Пример:

```js
fetch("https://api.example.com/data")
    .then(response => response.json())
    .then(data => console.log(data));
```

### **2. Зачем нужны `async/await`?**

`async/await` — это удобный синтаксис для работы с асинхронным кодом и промисами. Они позволяют писать асинхронный код так, будто он синхронный.

Преимущества:

* упрощают чтение кода,
* заменяют длинные цепочки `.then()`,
* делают обработку ошибок проще через `try...catch`.

Пример:

```js
async function getData() {
    try {
        const res = await fetch("https://api.example.com/data");
        const data = await res.json();
        console.log(data);
    } catch (err) {
        console.log("Ошибка", err);
    }
}
```

### **3. Как работает `localStorage`?**

`localStorage` — это встроенное хранилище браузера, куда можно сохранять данные в виде ключ–значение. Данные не удаляются после перезагрузки страницы или закрытия браузера.

Особенности:

* хранит только строки (все остальные типы нужно преобразовывать через JSON),
* объём примерно 5–10 МБ,
* данные сохраняются **навсегда**, пока пользователь сам не очистит.

Основные методы:

```js
localStorage.setItem("theme", "dark");
localStorage.getItem("theme");             
localStorage.removeItem("theme");          
localStorage.clear();                       
```

Применение в лабораторной работе:

* сохранение выбранной темы (light/dark),
* восстановление темы после перезагрузки.
